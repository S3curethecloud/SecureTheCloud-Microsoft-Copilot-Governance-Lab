# Control Catalog Workspace

**Workspace ID:** `control_catalog`  
**Phase:** 4B  
**Mode:** Static simulation shell

## Purpose

The Control Catalog Workspace records the control domains represented by the lab and gives the future platform a stable navigation surface for control evidence.

## Initial categories

- Identity Controls
- Conditional Access
- Purview Controls
- DLP Controls
- Data Protection Controls
- Copilot Governance Controls
- Audit Controls
- Adoption Controls

## Inputs

- control matrix
- synthetic control results
- generated evidence summaries
- downstream doctrine boundary records

## Outputs

- static control catalog documentation
- future control navigation metadata

## Boundary

This workspace is documentation and simulation evidence only. It has no runtime authority and does not connect to, configure, or enforce anything in live environments.
