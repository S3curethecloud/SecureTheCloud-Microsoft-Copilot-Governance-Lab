# Evidence Workspace Shell

**Phase:** 2 — Platform Readiness Architecture / Evidence Workspace Shell  
**Mode:** Static simulation shell

## Purpose

The evidence workspace shell is a static, generated view of synthetic readiness evidence. It gives the future platform a navigable evidence model without introducing backend services, user sessions, tenant connectivity, or live data access.

## Workspace sections

| Section | Purpose |
|---|---|
| Overview | Summarizes lab, authority, adoption status, and posture. |
| Navigation | Defines future workspace navigation anchors. |
| Control Explorer | Groups controls by domain and records pass/fail counts. |
| Failed Findings | Lists failed controls and remediation recommendations. |
| Evidence Artifacts | Lists generated artifacts and whether they are synthetic. |
| Claims Boundary | Records what the workspace does not claim or authorize. |

## Generated artifacts

Phase 2 adds two workspace artifacts:

- `workspace_index.json`
- `evidence_workspace.md`

Both are generated by:

```bash
python -m stc_copilot_lab.cli evaluate --fixtures data/fixtures --out evidence/generated --workspace
```

## Static shell guarantees

The workspace shell:

- uses synthetic fixture data only
- has no Microsoft tenant connection
- has no backend API
- has no user identity or session model
- has no authorization or RBAC behavior
- has no production enforcement
- has no real customer data
- makes no SOC 2 certification claim
